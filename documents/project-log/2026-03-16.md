# 2026-03-16 개발 기록

## 🛠️ 작업 내용
* **카테고리 기반 메뉴 옵션 템플릿(Template) 방식**:
  - 메뉴 등록 시 해당 카테고리에 설정된 기본 옵션들이 `MENU_OPTION_GROUP` 매핑 테이블에 자동으로 일괄 적용/복제되는 단일 복제 프로세스 도입.
  - 관리자가 일일이 옵션을 추가할 필요 없이 카테고리 지정만으로 공통 옵션을 신속 배포 가능.
* **메뉴별 전용 옵션 예외 처리 및 오버라이딩 (Override)**:
  - 특정 에이드 음료의 '온도' 옵션 같이 템플릿 상속 중 불필요한 공통 옵션을 개별 삭제할 수 있는 `removeOptionGroupFromMenu` 기능 연동.
  - 메뉴 조회 시 메뉴별 전용 옵션이 있으면 우선 적용하고, 없을 경우에만 카테고리 기본값을 불러오는 Fallback 조회 쿼리 완성.
* **주문 상세 옵션 추적 및 조회 (E2E Option Tracking)**:
  - 주문 내역 조회 시 `ORDER_ITEM_OPTION` 테이블을 JOIN하여 사용자가 주문 시점에 선택했던 샷 추가, ICE 등 실제 옵션 정보를 가져오도록 `OrderRepositoryImpl` 개선.
  - 회원 주문 내역 및 관리자 주문 관리 UI에서 메뉴 하단에 `▶ [선택된 상세 옵션: ...]` 형태로 상세하게 표시되도록 연동.
* **회원 관리 시스템 시각화 고도화**:
  - 가입된 회원들의 목록 조회 시 `MEMBER` 테이블의 가입일자(`created_at`) 정보를 함께 출력하도록 수정.
  - UI 레이아웃을 `ID | 전화번호 | 보유 포인트 | 등급 | 가입일자` 형태의 표 형식으로 일괄 개편하여 가독성 강화.
* **보안 및 예외 검증**:
  - 새로운 옵션 그룹을 생성할 때 기존의 그룹명과 중복되는 것을 실시간 체크하여 예외 차단 로직 추가.
  - 회원 주문 시 주문 상태(`COMPLETED`, `CANCELLED`)를 `[정상결제]`, `[취소완료]`로 한글화하여 직관성 증대.
* **팀 프로젝트 개발 내용 병합 및 버그 수정**:
  - **주문자 식별 오류 최종 해결**: 회원 주문 시 비회원으로 매핑되던 오류를 명시적 컬럼 매핑(Explicit Column Mapping) 및 세션 DTO 연동 강화로 해결.
  - 회원가입 시 선호 카테고리를 설정해도 1,000P 웰컴 보너스가 정상 적립되도록 예외 보강.
  - 런타임 중 `POINT_HISTORY` 테이블의 존재 여부를 자동 확인하고 없을 시 생성해주는 자동 구축 쿼리(`CREATE TABLE IF NOT EXISTS`) 탑재.
